# retro Skill Evaluation

## Test Case 1: Post-Launch Retro
**Input:** "Run retro for our Q2 product launch. We missed revenue targets by 30%."
**Expected Output:**
- Structured retro format (Start/Stop/Continue or similar)
- Cross-functional input capture
- Root cause analysis of miss
- Prioritized action items with owners
- Decision log

## Test Case 2: Sprint Retro
**Input:** "Retro for last 2-week sprint. Team feels we're shipping features without proper messaging."
**Expected Output:**
- Pattern identification across sprints
- Process gaps highlighted
- Recommended changes to handoff process
- Action items with accountability

## Test Case 3: GTM Cycle Review
**Input:** "Full GTM retro for FY24. What did we learn across all launches?"
**Expected Output:**
- Year-long pattern analysis
- Win/loss themes
- Process improvements for next year
- Compound learning documentation
