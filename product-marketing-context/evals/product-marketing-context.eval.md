## Folder Organization Logic

**Why organize by function, not skill count?**

- `pmm-foundation/` — Stores the brain setup skill + templates. Brain is the gravity well. It lives first.
- `pmm-meta/` — Meta skills (already have this). Verify, review, learn. These enforce quality.
- `pmm-segmentation/` — Beachhead + related targeting skills.
- `pmm-positioning/` — Positioning + messaging skills.
- `pmm-execution/` — Launch execution (GTM strategy, pre-mortem, experiment-doc, interview-summary).
- `pmm-competitive/` — Competitive analysis (battlecards, alternatives).
- `pmm-organization/` — Messaging, personas, voice, value props.

**This mirrors how PMM operators actually work:**
1. Build brain (foundation)
2. Pick a segment (segmentation)
3. Position against alternatives (positioning)
4. Plan launch (execution)
5. Analyze competitive landscape (competitive)
6. Define messaging & personas (organization)

## File Locations & Conventions

### Brain File (User-Facing)
```
/foundation/brain.md
```
Generated by `pmm-foundation/product-marketing-context/SKILL.md`.
Read by ALL other skills.

### Brain Template (Developer-Facing)
```
pmm-foundation/templates/brain-template.md
```
Used by product-marketing-context skill to populate and generate `/foundation/brain.md`.

### Evals Files (Testing)
```
pmm-foundation/evals/product-marketing-context.eval.md
pmm-positioning/evals/positioning-messaging.eval.md
pmm-execution/evals/pre-mortem.eval.md
[etc.]
```
Each folder has an `evals/` subdirectory with test cases for that skill.

### Skill-Specific README
```
pmm-foundation/README.md
pmm-positioning/README.md
[etc.]
```
Explains:
- What skills are in this folder
- Tier (T1/T2/T3/T4)
- Dependencies (which brain sections they read)
- Quick start (what to run first)

### Root-Level Files
```
SKILL-SPEC.md         ← Scoring standard (all skills measured against this)
README.md             ← Repo overview ("what is this?")
SETUP.md              ← User onboarding ("how do I get started?")
```

## Why This Structure?

1. **Brain is first** — It's in pmm-foundation, the first folder. Users read SETUP.md, run product-marketing-context first, everything else follows.

2. **Organized by function** — pmm-positioning skills are together, pmm-execution skills are together. Users find what they need by category.

3. **Meta skills are separate** — pmm-meta is its own universe. It enforces quality but doesn't interfere with domain skills.

4. **Templates are central** — brain-template.md is in pmm-foundation/templates, visible to everyone. Easy to update when brain structure changes.

5. **Evals are local** — Each folder has evals/. Tests stay close to code. Easy to maintain.

6. **/foundation/ is runtime** — User's brain lives at `/foundation/brain.md`. No versioning, no folders. Simple. Clean.

7. **Root docs are clear** — SETUP.md, README.md, SKILL-SPEC.md at root. Users know where to start.
